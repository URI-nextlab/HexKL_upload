Enumeration of supported tensor data types in SDKL.

This enum defines the scalar types that tensors can use for storage and computation. These types include integer, unsigned integer, and floating-point formats with varying precision.

enum sdkl_tensor_layout_e

Enumeration of supported tensor memory layouts in SDKL.

This enum defines the memory layout formats for tensors in SDKL. It includes options for 1D and 2D tensors, with variants for standard and HMX-specific layouts.

Enumerator

1D tensor with linear memory layout.

SDKL_LAYOUT_2D_ROW_MAJOR 

2D tensor with row-major layout (C-style).

SDKL_LAYOUT_2D_COL_MAJOR 

2D tensor with transposed layout (column-major).

SDKL_LAYOUT_2D_ROW_MAJOR_WEIGHTS_HMX 

2D tensor with row-major layout for weights matrix optimized for the HMX unit.

SDKL_LAYOUT_2D_ROW_MAJOR_ACTIVATION_HMX

enum sdkl_tensor_platform_e

Enumeration of supported execution platforms for tensor operations in SDKL.

This enum defines the available hardware platforms on which tensor operations can be executed. Selection of platform may depend on performance, power efficiency, or hardware availability.

Note
The number and identity of NPUs (e.g., NPU0, NPU1) are platform-dependent and may vary across Snapdragon devices. It is up to the programmer to know how many NPUs are available.
The beta version of SDKL supports only NPU0. Future releases may add support for additional platforms
Enumerator
SDKL_PLATFORM_CPU 

Execute tensor operations on the CPU.

SDKL_PLATFORM_GPU 

Execute tensor operations on the GPU (Graphics Processing Unit).

SDKL_PLATFORM_NPU0 

Execute tensor operations on the NPU0 (Neural Processing Unit #0).

SDKL_PLATFORM_NPU1 

Execute tensor operations on the NPU1 (Neural Processing Unit #1).

SDKL_PLATFORM_INVALID 

Invalid platform selection. Also used to indicate the number of valid values.
