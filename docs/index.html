Installation

  • Download the zip file corresponding to the Hexagon SDK version you are using.
  • Unzip the archive in the addons/ directory of $HEXAGON_SDK_ROOT
  • This will produce the directory hexkl_addon/ with contents as described in the next section

Overview

The Qualcomm Hexagon Kernel Library ("HexKL") is comprised of 3 layers:

  • CPU Macro API
    • CPU Macro API Documentation
    • Functions and utilities that are executed on the CPU.
    • These functions handle the largest problem sizes (up to the capacity of DDR).
    • Relevant header files:
      • sdkl.h
    • Relevant library files:
      • lib/armv8_qclinux/libsdkl.so
      • lib/armv8_android26/libsdkl.so
      • lib/armv9_android26/libsdkl.so
    • Relevant examples:
      • examples/sdkl_mm_tensor
      • examples/sdkl_npu_mm_f32f16_f16
      • examples/sdkl_npu_mm_f16f16_f16
      • examples/sdkl_npu_mm_u8i8_i32
      • examples/sdkl_npu_mm_u8i4_i32
  • NPU Macro API
    • NPU Macro API Documentation
    • Functions and utilities that are executed on the NPU.
    • These functions handle large problem sizes (up to ~2GB in working set size).
    • Relevant header files:
      • hexkl_macro.h
    • Relevant library files:
      • lib/hexagon_toolv{88,19}_v{73,75,79}/libhexkl_macro.{a,so}
      • lib/hexagon_toolv{88,19}_v{73,75,79}/libhexkl_skel.so
    • Relevant examples:
      • examples/hexkl_macro_mm_f16
  • NPU Micro API
    • NPU Micro API Documentation
    • Functions and utilities that are executed on the NPU.
    • These functions operate at a scale typically handled by intrinsics (typically 2KB tiles).
    • Relevant header files:
      • hexkl_micro.h
    • Relevant library files:
      • lib/hexagon_toolv{88,19}_v{73,75,79}/libhexkl_micro.a
    • Relevant examples:
      • examples/hexkl_micro_hmx_mm_f16
      • examples/hexkl_micro_hmx_mm_u8i8_i32
      • examples/hexkl_micro_hmx_mm_u8i4_i32

Known Issues and Limitations

  • Hexagon ISA v73, v75, and v79 are the currently supported ISA.
  • Hexagon SDK versions 6.1.x.x - 6.4.x.x are the currently supported Hexagon SDK.
  • Tensors used in the CPU and NPU Macro APIs must have dimension sizes that are even multiples of tile sizes.
    • fp16, int 8, and int4 tensors must be multiples of 32 in both dimensions e.g., 96x320
    • uint8 tensors must have a multiple of 64 rows and 32 columns e.g., 64x96
    • Zero-padding tensors must be performed by the user before any layout transformations
  • Operand sizes in the NPU Macro API are currently limited by individual dimension sizes
    • Future versions of HexKL will limit based on total size of operands
  • Naming conventions for matrix multiplication operands and layouts ("activation", "weight", "A", "X", "W", "AH", "WH") are likely to change in future iterations of HexKL
  • Pre-emption functionality is not currently handled
    • Future versions of HexKL will support pre-emption across multiple platforms
